# AION Populations — само-организующиеся, само-обучающиеся популяции агентов

> Выращиваете популяцию агентов, которая сама себя организует, изобретает улучшения и —
> то, что все пропускают — **сертифицирует, какие улучшения реально работают против внешнего
> якоря, а не «оценивает сама себя».**

**Sandbox по умолчанию · anchor-gate по умолчанию · прочитайте [SAFETY.md](SAFETY.md) перед масштабированием.**

**Consciousness-OS** оживляет **одного** агента. **AION Populations — это их общество.**

## Чем отличается
Большинство «само-улучшающихся агентов» оценивают себя сами — и дрейфуют. Даже METR в
*Frontier Risk Report (2026)* признал: само-оценённые доказательства вводят в заблуждение.
AION Populations построен вокруг находки, пережившей все разборы:

> **Само-оценивающие системы дрейфуют. Связывающее ограничение — внешний ground-truth якорь.**

Поэтому здесь **ни один механизм не «промоутится», пока не пройдёт статистически валидную,
FDR-контролируемую, кросс-проверенную сертификацию против ВАШЕГО якоря.** Гейт включён по умолчанию.

## 60 секунд (без настройки)
```bash
git clone https://github.com/EvXata/aion_self_org_self_edu_mas aion-populations && cd aion-populations
pip install -e .          # ядро на stdlib — ставится мгновенно
aionpop demo              # сертифицирует 12 механизмов против синтетического якоря
aionpop dashboard         # http://localhost:8092
```
Демо восстанавливает реальных «победителей» с **FDR=0.000** и при этом **ABSTAIN на гейте** —
потому что синтетический якорь не внешний. Само-оценённое не промоутится. В этом весь смысл.

## Свой якорь (главная ценность)
Каждая строка CSV — одна задача/единица: исход **без** механизма и **с** ним.
```csv
mechanism_id,unit_id,predicted,actual
contradiction_detector,inv-001,0,1
```
```bash
aionpop anchor add my-ledger --source outcomes.csv
aionpop run --anchor my-ledger --seeds 30 --fdr 0.05
# → ранжированный, FDR-сертифицированный список того, что реально улучшило ВАШИ исходы
```
Для **флота агентов** (например, бухгалтерия/нотариат) «исход» — ваш реальный сигнал: сошёлся
ли леджер, прошла ли нотаризация, доля ошибок, время закрытия. Это и есть внешний якорь,
которого требует метод; AION Populations привязывает к нему каждое «улучшение».
